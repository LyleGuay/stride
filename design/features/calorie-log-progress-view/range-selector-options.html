Progress Range Selector — Design Options

Each option shown in context with the tab bar above it. Active selection: Last 6 Mo.

Option A — Floating Pills

Auto-width rounded pills, no container box. Distinct from the tab bar above. Clean, minimal.

chart
Option B — Underline Tabs

Text-only with a colored underline. Common in analytics dashboards (Google Analytics, Notion). Very lightweight — no boxes at all.

chart
Option C — Accent Pills (stride color)

Same pill shape as Option A but uses the app's indigo accent color for the active state. More visually prominent.

chart
Option D — Dropdown

Keeps the UI clean by tucking the selector into a single compact control. Leaves max space for the chart. Good if options ever grow further.

Showing
chart
Option E — Pills with Granularity Hint

Each pill shows a tiny sub-label indicating bar granularity (days / weeks / months). Gives the user a preview of what the chart will look like before clicking.

chart
Option F — Grouped Clusters (short-term / long-term)

Two pill clusters separated by a divider, making the short-term vs. long-term grouping explicit. Labels above each cluster.

chart
Option G — Inline with Chart Header

The range selector moves inside the chart card header, right-aligned next to "Calories". No separate row at all — saves vertical space.

Option H — Pills with Date Range Preview

Active pill shows the date range it covers as a sub-label. Only the selected pill shows the range — keeps it minimal until needed.
